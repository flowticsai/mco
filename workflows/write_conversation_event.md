# Write Conversation Event

**n8n ID:** `5qOo5YzrPnW8Uj9g`  
**Entry:** `POST /webhook/mco-write-event`  
**Node count:** 21 | **Status:** Active

## Objective
Record every lead interaction across all channels (email, LinkedIn, voice, SMS) into the unified Supabase store and Notion CRM. This is the single write path — all channel workflows call this instead of writing to storage themselves.

## When to Use
Call this immediately after any of the following:
- An inbound message is received from a lead (any channel)
- An outbound message is sent to a lead (any channel)
- A voice call completes (Retell AI post-call webhook)
- A follow-up is dispatched from the FollowUpQueue
- A LinkedIn connection request is sent (from Coordinator)
- A LinkedIn connection is accepted (from Connection Accepted Handler)

## Lead Identity — at least one identifier required

| Identifier | Notes |
|---|---|
| `lead_id` | UUID — preferred when you already have it (fastest lookup) |
| `lead_email` | Lowercase. Always provide if known |
| `linkedin_profile_url` | Full URL e.g. `https://linkedin.com/in/johndoe` |
| `linkedin_urn` | Aimfox-format URN e.g. `urn:li:person:AbCdEfGhIj` |
| `phone_e164` | E.164 format e.g. `+15551234567` |

## All Input Fields

| Field | Required | Notes |
|---|---|---|
| `event_id` | YES | UUID v4 generated by caller. Same ID on retry = idempotent (safe to retry) |
| `channel` | YES | `email` \| `linkedin` \| `voice` \| `sms` |
| `direction` | YES | `inbound` \| `outbound` |
| `content` | YES | Message text, transcript, or summary |
| `timestamp` | YES | ISO 8601 UTC. When the interaction occurred (not when recorded) |
| `lead_email` | one of these | Lowercase |
| `lead_id` | one of these | UUID from leads table |
| `linkedin_profile_url` | one of these | Full LinkedIn URL |
| `linkedin_urn` | one of these | Aimfox URN |
| `phone_e164` | one of these | E.164 format |
| `content_type` | no | `message` \| `transcript` \| `summary`. Default: `message` |
| `intent` | no | Intent classification for this interaction. Default: `unknown` |
| `sender_name` | no | Display name of the sender |
| `full_name` | no | Lead full name (used on first write to leads table) |
| `company` | no | Lead company name |
| `linkedin_conversation_urn` | no | Stored on lead record for future LinkedIn DM replies |
| `aimfox_campaign` | no | Campaign ID if this lead was added to an Aimfox campaign |
| `metadata` | no | JSON object: any extra context (call_id, campaign_id, queue_id, etc.) |
| `workflow_execution_id` | no | n8n execution ID for debugging |
| `trigger_cross_channel` | no | `true` to queue follow-ups on other channels |
| `target_channels` | no | Array e.g. `["email"]` — which channels to queue |
| `follow_up_context` | no | Short summary passed to the follow-up agent |

## Node Sequence

1. **Webhook** — `POST /mco-write-event`
2. **Setup & Validate** — Code node
   - Requires at least one lead identifier
   - Requires: event_id, channel, direction, content, timestamp
   - Normalises lead_email to lowercase
   - Defaults content_type to `message`, intent to `unknown`
3. **Upsert Lead** — Supabase `upsert_lead()` RPC
   - Multi-identifier resolution: matches by any identifier, creates if none match
   - Atomic intent promotion (never demotes)
   - Returns: `lead_id`, `lead_email`, `lead_created`, `overall_intent`
4. **Merge Lead Data** — Code node, merges RPC response back into the payload
5. **Insert Conversation** — Supabase `insert_conversation_event()` RPC
   - Passes `lead_id` (preferred) or `lead_email`
   - Dedup on `event_id` — safe to call even after a retry
6. **Check Insert Result** / **Was Duplicate?** — stops early and returns 200 if `was_new: false`
7. **Has Phone?** → **Upsert PhoneMap** — Supabase REST, table `phone_map`, ON CONFLICT DO NOTHING
8. **Notion: Query Lead** → **Notion: Extract Lead ID** → **IF: Notion Lead Found?**
   - Queries Notion database `362dc227-748f-8184-892a-c6f8f3151b07` by `lead_email`
   - **Found:** Notion: Update Lead (PATCH /pages/:id with latest intent, name, company)
   - **Not found:** Notion: Create Lead (POST /pages — new lead page)
   - Both paths → **Notion: Create Conversation** (POST /pages — new conversation entry)
9. **Cancel Pending On Inbound** — Code node
   - If `direction = inbound` and a lead is identified: reschedules any `pending` follow_up_queue items for this lead to avoid sending a follow-up when the lead just replied
10. **Cross-Channel?** → **Build Queue Rows** → **Insert FollowUpQueue** — if `trigger_cross_channel: true`
    - For each channel in `target_channels`: inserts row with `status=pending`, `scheduled_for=NOW()+30min`
    - Includes `lead_id` so Dispatcher can pass it to the Coordinator
11. **Return OK** — 200 with `{ status, lead_id, lead_email, lead_created, overall_intent }`

## Output
```json
{
  "status": "ok",
  "lead_id": "uuid-here",
  "lead_email": "john@acme.com",
  "lead_created": false,
  "overall_intent": "interested"
}
```

## Error Handling
- **Missing identifier** → 400
- **Missing required fields** → 400
- **Duplicate event_id** → 200, `{ status: "already_written" }`
- **Supabase write failure** → let n8n retry (event_id dedup makes retries safe)

## Notes
- `upsert_lead()` handles race conditions atomically. Never implement intent promotion in n8n Code nodes.
- `trigger_cross_channel` should only be `true` when intent = `interested` or `booking`.
- `linkedin_conversation_urn` is stored on the lead record so the Coordinator can always find the right conversation thread for LinkedIn DMs without a separate lookup.
- Notion integration runs after the Supabase write — Notion failure does not prevent the 200 response.
- `Cancel Pending On Inbound` ensures we don't double-send: if the lead replies, pending follow-ups are pushed back automatically.
