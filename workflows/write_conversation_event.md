# Write Conversation Event

## Objective
Record every lead interaction across all channels (email, LinkedIn, voice, SMS) into the unified Supabase store and sync to Notion CRM. This sub-workflow is the single write path — all channel workflows call this instead of writing to storage themselves.

## When to Use
Call this immediately after any of the following:
- An inbound message is received from a lead (any channel)
- An outbound message is sent to a lead (any channel)
- A voice call completes (Retell AI post-call webhook)
- A follow-up is dispatched from the FollowUpQueue

## Required Inputs
All fields sent as JSON body to the workflow webhook or via Execute Workflow node:

| Field | Required | Notes |
|-------|----------|-------|
| event_id | YES | UUID v4 generated by caller. Same ID on retry = idempotent (safe to retry) |
| lead_email | YES | Lowercase. Placeholder format: `urn_{slug}@linkedin.placeholder` |
| timestamp | YES | ISO 8601 UTC. When the interaction occurred (not when recorded) |
| channel | YES | `email` \| `linkedin` \| `voice` \| `sms` |
| direction | YES | `inbound` \| `outbound` |
| content | YES | Message text, transcript, or summary |
| content_type | YES | `message` \| `transcript` \| `summary` |
| intent | YES | Current intent classification for this interaction |
| sender_name | no | Display name of the sender |
| metadata | no | JSON object: thread_id, call_id, aimfox_convo_id, etc. |
| workflow_execution_id | no | n8n execution ID (for debugging) |
| phone_e164 | no | Include if known; written to phone_map table |
| full_name | no | Lead full name (used on first write to leads table) |
| company | no | Lead company name |
| linkedin_urn | no | Aimfox URN if available |
| trigger_cross_channel | no | `true` to queue follow-ups on other channels |
| target_channels | no | Array: `["email","sms"]` — which channels to queue follow-up on |
| follow_up_context | no | Short summary passed to follow-up agent as context |

## Node Sequence

1. **Webhook Entry** — POST `/mco-write-event`
2. **Validate & Normalize** — Code node
   - Require: event_id, lead_email, channel, direction, content, timestamp
   - Normalize lead_email to lowercase
   - Strip whitespace from phone_e164; validate E.164 format if present
   - Default content_type to `message` if omitted
   - Return error 400 if required fields missing
3. **Dedup Check** — Supabase SELECT
   - Query: `SELECT event_id FROM conversations WHERE event_id = '{{event_id}}'`
   - If row found → respond 200 `{ status: "already_written" }` and stop
4. **Resolve/Create Monday Item** — Monday.com GraphQL HTTP Request
   - Search board `18399476470` for item where email column = lead_email
   - If not found → create new item with lead name as title
   - Store returned monday_item_id
5. **Upsert Lead** — Supabase RPC call to `upsert_lead()` function
   - Pass all available lead fields
   - Function handles intent promotion atomically (never demotes)
   - Returns: lead_created (bool), monday_item_id, current overall_intent
6. **Insert Conversation** — Supabase INSERT
   - Table: `conversations`
   - ON CONFLICT (event_id) DO NOTHING (belt-and-suspenders after dedup check)
7. **Upsert PhoneMap** — Supabase INSERT (only if phone_e164 present)
   - Table: `phone_map`
   - ON CONFLICT (phone_e164) DO NOTHING
8. **Post Monday Update** — Monday.com "Add update to item"
   - Format: `{emoji} {CHANNEL} | {Direction} | {timestamp UTC}\nFrom: {sender_name}\n───────\n{content (max 500 chars)}\nIntent: {intent}`
   - Emojis: 📧 email · 💼 linkedin · 📞 voice · 💬 sms
9. **Update Monday Item Columns** — Monday.com column values mutation
   - Set: overall_intent, last_active_channel, phone_e164 (if present)
10. **Write FollowUpQueue** — Supabase INSERT (only if trigger_cross_channel = true)
    - For each channel in target_channels: insert row with status=pending, scheduled_for=NOW()+30min
11. **Return Response** — 200 OK
    - `{ status: "ok", monday_item_id: "...", lead_created: true/false, overall_intent: "..." }`

## Outputs
```json
{
  "status": "ok",
  "monday_item_id": "12345678",
  "lead_created": false,
  "overall_intent": "interested"
}
```

## Error Handling
- **Missing required fields** → 400 with field list
- **Supabase write failure** → let n8n retry (event_id dedup makes retries safe)
- **Monday.com API failure** → log to Slack `#system-errors` but do NOT fail the whole workflow (Monday is secondary to Supabase)
- **Phone E.164 validation failure** → skip phone_map write, continue with rest of flow

## Notes
- The `upsert_lead()` Postgres function handles race conditions atomically. Do not try to implement intent promotion logic in the n8n Code node.
- `trigger_cross_channel` should only be `true` when intent = `interested` or `booking` — not for every event.
- content is truncated to 500 chars in the Monday.com update only; full content is always stored in Supabase.
